# To PRD Command

Use this workflow when the user wants to plan a new feature for MyOrganizer — writing a PRD and publishing it as a GitHub Issue.

1. Read and follow `.github/skills/to-prd/SKILL.md` exactly.
2. The user must be present — this skill has one interactive step (test seam approval) before the PRD is written.
3. Run `yarn ai:create-labels` first if any required labels are missing from the repo.
