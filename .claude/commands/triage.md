# Triage Command

Use this workflow when triaging GitHub issues or external PRs into actionable states.

1. Read and follow `.github/skills/triage/SKILL.md` exactly.
2. Use `.github/skills/triage/AGENT-BRIEF.md` when moving work to `ready-for-agent`.
3. Use `.github/skills/triage/OUT-OF-SCOPE.md` when rejecting enhancements as `wontfix`.
4. Start every triage-generated issue comment with the required AI disclaimer from the skill.
5. Treat PR triage as issue triage with attached code.
