# `@myorganizer/design-tokens`

Single source of truth for the MyOrganizer design system (colors, spacing,
typography, radius, shadow). Based on the **Secure Modernism** spec in
[DESIGN.md](DESIGN.md).

## Files

- [src/tokens.json](src/tokens.json) — **edit this.** W3C
  [Design Tokens Community Group format](https://tr.designtokens.org/format/).
- `src/generated/` — **do not edit.** Regenerated by Style Dictionary.
  - `tokens.css` — CSS custom properties (`:root { --color-primary: … }`).
  - `tokens.ts` — typed TS constants + `var(--…)` strings.
  - `tailwind-preset.js` — Tailwind preset; consumed by
    [apps/myorganizer/tailwind.config.js](../../apps/myorganizer/tailwind.config.js).

## Workflow

1. Edit `src/tokens.json`.
2. Regenerate: `yarn nx run design-tokens:build-tokens`.
3. Commit both `tokens.json` and `src/generated/`.

## Consuming tokens

**Tailwind classes (preferred for components):**

```tsx
<button className="bg-primary text-on-primary rounded-md shadow-card">…</button>
```

**Raw CSS:**

```css
.thing {
  background: var(--color-primary);
}
```

**TypeScript (when a value must reach JS, e.g. SVG fill):**

```ts
import { colorPrimary, colorPrimaryVar } from '@myorganizer/design-tokens';
```

## Rules

- **Never hard-code hex colors, spacing values, or font stacks in
  components.** Use a Tailwind class or import from
  `@myorganizer/design-tokens`.
- New token? Add it to `tokens.json` first, regenerate, then use it.
- Keep DESIGN.md as the human-facing rationale; this lib is the machine
  truth.
